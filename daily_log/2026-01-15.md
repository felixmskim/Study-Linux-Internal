## 🔍 소스코드 분석: `goto` 에러 처리 패턴
- **석 대상:** `One err bugs` 방지를 위한 순차적 리소스 해제
- **핵심 코드:**
```C
// 할당 역순으로 label을 배치하여 NULL 포인터 역참조 방지
err_free_bar:
    kfree(foo->bar);
err_free_foo:
    kfree(foo);
    return ret;
```
단순히 에러 지점으로 점프하는 것이 아니라, 리소스가 할당되었는지를 고려하여 레이블을 세분화해야 함.
